**Key constraint (Phase 1 decision):** `view` defaults to `spielplan`. Existing embeds must not break. The generator MUST omit `&view=` when spielplan is selected — do NOT append `&view=spielplan`. Only append `&view=table` (or any non-default value).

**URL building location in generator.html:** Inside the `generate-btn` click handler (line ~319). Params are built as:
```js
var params = teams.map(function(t) { return 't=' + t; }).join('&') + '&color=' + color;
if (pastCount  !== 3)  params += '&past='        + pastCount;
if (futureCount > 0)   params += '&future='      + futureCount;
if (!showPast)         params += '&show_past=0';
if (!showFuture)       params += '&show_future=0';
if (!showTitle)        params += '&title=0';
if (compact)           params += '&compact=1';
var src    = base + '?' + params;
```
Add the view param in the same conditional pattern, after the `compact` line and before `var src`.
