<feature>
  <name>league-config validator and loader</name>
  <files>league-config.js, tests/league-config.test.js</files>
  <behavior>
    validateLeagueConfig(config) — pure validator, no I/O:
    1. null or non-object input → throws Error('league-config: config must be a non-null object')
    2. empty object {} → throws Error('league-config: no leagues defined')
    3. season entry missing 'name' field → throws Error("league-config: ff-bl/2026 missing field 'name'")
    4. season entry with gameday_ids not an array → throws Error('league-config: ff-bl/2026 gameday_ids must be an array')
    5. season entry with non-integer in gameday_ids → throws Error('league-config: ff-bl/2026 gameday_ids must contain only integers')
    6. season entry with non-integer in promotion_restricted → throws Error('league-config: ff-bl/2026 promotion_restricted must contain only integers')
    7. fully valid config → returns the same config object (no mutation)

    loadLeagueConfig(filePath) — reads file, parses JSON, delegates to validateLeagueConfig:
    8. valid file (league-config.json created by 03-01) → returns validated config object
       (file-not-found and JSON parse errors propagate naturally — no wrapping needed)
  </behavior>
  <implementation>
    Create `league-config.js` as a CommonJS module at project root:

    - `validateLeagueConfig(config)`:
      - Guard: if config is null or typeof !== 'object' → throw
      - Guard: if Object.keys(config).length === 0 → throw
      - For each league key, for each season key:
        - Check 'name' is present and typeof === 'string'
        - Check 'gameday_ids' is an Array; every element passes Number.isInteger()
        - Check 'promotion_restricted' is an Array; every element passes Number.isInteger()
        - Use error message format: `"league-config: {league}/{season} {message}"`
      - Return config (identity)

    - `loadLeagueConfig(filePath)`:
      - `const raw = require('fs').readFileSync(filePath, 'utf8');`
      - `const config = JSON.parse(raw);`
      - `return validateLeagueConfig(config);`

    - `module.exports = { validateLeagueConfig, loadLeagueConfig };`

    Wire into `_gen_snapshot.js`:
    - Add `const { loadLeagueConfig } = require('./league-config.js');` near top (after the constant declarations)
    - Add `const leagueConfig = loadLeagueConfig('./league-config.json');` as first statement inside the async IIFE (before any fetch calls)
    - Add `console.log('League config loaded: ' + Object.keys(leagueConfig).join(', '));`
  </implementation>
</feature>
