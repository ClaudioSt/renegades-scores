**Established patterns:**
- `renderGamedayCard(gd, games, teamId, showScore)` returns HTML string — study its structure for field access
- `renderGameRow(game, teamId, showScore)` renders a single game row — reuse or adapt for the highlight card
- All user/API data goes through `escapeHtml()` before innerHTML (security requirement from CLAUDE.md)
- `--accent` CSS custom property is set on `:root` from `cfg.color`
- Widget uses `var` declarations in widget.html (ES5 style)
- CSS class naming: kebab-case (`.next-game-highlight`, `.next-game-date`, etc.)

**findNextGame API (from 02-03):**
```js
findNextGame(activeEntries, teamId, today)  // returns {gd, game} or null
```

**Where to display:**
- In `loadTeam()`, in the future section rendering block (~line 631)
- The highlight card goes BEFORE the regular future gameday cards (it IS the next game; it appears as a distinct visual element, not a duplicate card)
- If `findNextGame` returns null: render nothing extra (no empty placeholder)
- If `showFuture` is false (cfg.showFuture === false): skip entirely

**Card content spec:**
- Gameday name: `gd.name`
- Date: formatted with `formatDate(gd.date)` (existing function)
- Time: from `game.scheduled` if present (extract HH:MM), else omit
- Opponent: the result entry where `team_id !== teamId` — use `escapeHtml(r.team_name)`
- Home/away indicator: `game.results.find(r => r.team_id === teamId).isHome` → "Heimspiel" / "Auswärtsspiel"
- No score shown (it's upcoming)

**HTML structure for the card (example):**
```html
<div class="next-game-highlight">
  <div class="next-game-label">Nächstes Spiel</div>
  <div class="next-game-date">[date] · [time]</div>
  <div class="next-game-name">[gameday name]</div>
  <div class="next-game-opponent">[home/away] · [opponent name]</div>
</div>
```
Keep it simple — 4 lines of info is enough. Adapt if the actual snapshot data suggests a better layout.

  <action>
    1. Add `function renderNextGameCard(gd, game, teamId)` as a top-level function in the RENDERING section (near renderGamedayCard / renderGameRow):

    ```js
    function renderNextGameCard(gd, game, teamId) {
      var me    = game.results.find(function(r) { return r.team_id === teamId; });
      var other = game.results.find(function(r) { return r.team_id !== teamId; });
      if (!me || !other) return '';
      var time = '';
      if (game.scheduled) {
        var t = game.scheduled.slice(11, 16);
        if (t) time = ' · ' + t + ' Uhr';
      }
      var location = me.isHome ? 'Heimspiel' : 'Auswärtsspiel';
      return '<div class="next-game-highlight">'
        + '<div class="next-game-label">Nächstes Spiel</div>'
        + '<div class="next-game-date">' + escapeHtml(formatDate(gd.date)) + time + '</div>'
        + '<div class="next-game-name">' + escapeHtml(gd.name) + '</div>'
        + '<div class="next-game-opponent">' + escapeHtml(location) + ' · ' + escapeHtml(other.team_name) + '</div>'
        + '</div>';
    }
    ```

    2. In `loadTeam()`, in the future/active rendering block (~line 631), prepend the highlight card to `els.future.innerHTML`:

    ```js
    if (els.future) {
      var futureEntries = activeEntries
        .filter(function(e) { return classifyGameday(e.gd, e.games, teamId, today) === 'active'; })
        .sort(function(a,b) { return a.gd.date.localeCompare(b.gd.date); });
      if (cfg.future > 0) futureEntries = futureEntries.slice(0, cfg.future);

      // Highlight card for next game (above regular cards)
      var nextResult = findNextGame(futureEntries, teamId, today);
      var highlightHtml = nextResult ? renderNextGameCard(nextResult.gd, nextResult.game, teamId) : '';

      els.future.innerHTML = highlightHtml + (futureEntries.length
        ? futureEntries.map(function(e) { return renderGamedayCard(e.gd, e.games, teamId, false); }).join('')
        : '<div class="no-data">Keine kommenden Spieltage geplant.</div>');
    }
    ```

    Note: if `cfg.showFuture` is false, `els.future` is null — the guard `if (els.future)` already handles that.
    All fields passed through `escapeHtml()` in renderNextGameCard. No raw API data in innerHTML.
