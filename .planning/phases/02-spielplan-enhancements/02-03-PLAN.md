**Established patterns:**

  - `gd.date`: ISO date string 'YYYY-MM-DD'
  - `game.scheduled`: ISO datetime or null
  - `game.status`: 'scheduled'|'live'|'final'|'postponed'

**Function signature to implement:**
```js
function findNextGame(activeEntries, teamId, today)
```
- `activeEntries`: array of `{id, gd, games}` (same as what loadTeam processes for future/active section)
- `teamId`: number
- `today`: ISO date string 'YYYY-MM-DD'
- Returns: `{gd, game}` object, or `null` if none found

**Selection logic:**
- Consider only entries where `gd.date >= today` (don't show past-dated gamedays)
- Within qualifying entries, filter games that have `results` containing this teamId
- Exclude games with `status === 'final'` (already played)
- Sort qualifying entries by `gd.date` ascending — pick the earliest
- Within that earliest gameday, pick the first game involving teamId
- Return `{gd: entry.gd, game}` or `null`

**Edge cases to cover in tests:**
1. Empty activeEntries → null
2. All entries have gd.date < today → null
3. Single future entry, single game → returns {gd, game}
4. Multiple entries → returns the earliest-dated one
5. Entry has games for other teams only → skipped, returns null
6. Entry has game with status 'final' → skipped
7. Entry has multiple games, team appears in both → returns first match
</context>
