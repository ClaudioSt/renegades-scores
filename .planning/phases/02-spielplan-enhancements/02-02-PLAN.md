**Established patterns (01-02 summary):**
- `renderSpielplan` is a top-level async function with access to `root` (the widget container div)
- `loadTeam(teamId, els, cfg)` is the main data-load-and-render function (~lines 519–640)
- `quickRenderFromSnap(snap, teamId, els, cfg, today)` is the quick-path renderer (~lines 490–518)
- Both currently use the `<details class="more-gamedays">` collapse in their past sections

**renderPastSection API (from 02-01):**
```js
renderPastSection(pastSorted, teamId, visibleCount)
```
Returns HTML string. `pastSorted` = array of `{id, gd, games}`.

**Click handler spec:**
- Delegate from `root` (event target with class `load-more`)
- Read `data-team` (parse as int — team ID) and `data-visible` (parse as int)
- Increment visible by 3: `newVisible = parseInt(btn.dataset.visible) + 3`
- Re-render the past container: `document.getElementById('t-past-' + teamId).innerHTML = renderPastSection(pastSorted, teamId, newVisible)`
- `pastSorted` must be accessible at click time — store on the button's closest `team-block` via `data-*` or closure

**State storage approach (closure preferred):**
- In `loadTeam()`, after building `pastSorted`, store it as a closure variable in a per-team Map or directly on the container element as a custom property (not data-* since arrays can't be encoded as strings without JSON)
- Recommended: use a `var _pastCache = {}` module-level map keyed by teamId, storing the sorted array; click handler reads from it

**CSS for the button:**
The `.load-more` button style — add it with the existing widget CSS. Minimal: full-width, subtle background, uses `--accent` color on hover. No new font or icon dependencies.

<task type="auto">
  <name>Task 2: Add click event delegation for "Weitere laden" button in renderSpielplan</name>
  <files>widget.html</files>
  <action>
    Inside `renderSpielplan(root, cfg)`, after appending all team blocks to root, add a single delegated click listener on `root`:

    ```js
    root.addEventListener('click', function(e) {
      var btn = e.target.closest('.load-more');
      if (!btn) return;
      var teamId    = parseInt(btn.getAttribute('data-team'), 10);
      var newVisible = parseInt(btn.getAttribute('data-visible'), 10) + 3;
      var pastEl = document.getElementById('t-past-' + teamId);
      if (!pastEl || !_pastCache[teamId]) return;
      pastEl.innerHTML = renderPastSection(_pastCache[teamId], teamId, newVisible);
    });
    ```

    Place the listener after the `await Promise.all(...)` call (after data has loaded and team blocks exist).

    `Element.closest()` is available in all modern browsers and is safe here (widget targets ES2020+).

    Do not add separate listeners per team block — one delegated listener on root handles all teams.
