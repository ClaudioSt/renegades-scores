  <behavior>
    parseConfigFromSearch() must return a `view` field:

    - absent        → view === 'spielplan'   (backwards compat — all existing embeds)
    - ?view=spielplan → view === 'spielplan' (explicit, same as default)
    - ?view=table   → view === 'table'       (new table view route)
    - ?view=ANYTHING → view === 'ANYTHING'   (pass-through; dispatch handles unknown → spielplan)
    - view param does not affect any other config fields (teams, color, past, etc.)

    Cases:
    parseConfigFromSearch('')          → { ..., view: 'spielplan' }
    parseConfigFromSearch('?t=159')    → { ..., view: 'spielplan' }

    parseConfigFromSearch('?view=spielplan')   → { ..., view: 'spielplan' }
    parseConfigFromSearch('?t=159&view=table') → { teams: [159], ..., view: 'table' }
  </behavior>
