## Naming Conventions

**Files:**
- kebab-case for workflows and documentation: `update-snapshot.yml`, `leaguesphere-api-analysis.md`
- Lowercase with dashes for HTML: `widget.html`, `generator.html`
- Underscore prefix for Node.js scripts: `_gen_snapshot.js` (signals "run locally or CI, not a web page")
- UPPERCASE for project docs: `README.md`, `CLAUDE.md`
- `*.test.js` for test files in `tests/`

**In-code naming:**
- `UPPER_SNAKE_CASE` — constants (`API_BASE`, `CACHE_VERSION`, `DISCOVERY_TTL_MS`)
- `camelCase` — functions and variables (`parseConfigFromSearch`, `escapeHtml`)
- `_underscore` prefix — private/internal globals (`_snapshotPromise`, `_openDetail`)
- `lsw_` prefix — localStorage keys (`lsw_{teamId}`)

## Where to Add New Code

**New widget feature (UI / behavior):**
- Implementation: inline `<script>` in `widget.html`
- Tests: `tests/unit.test.js` (pure logic) or `tests/security.test.js` (rendering)
- If function needs to be testable: ensure it's declared at module scope (not inside IIFE closure)

**New snapshot field:**
- Generator logic: `_gen_snapshot.js`
- Consumer: `widget.html` or `generator.html`
- Validation test: `tests/snapshot.test.js`
- Bump `CACHE_VERSION` in `widget.html` if localStorage schema changes

**New URL parameter:**
- Parse: `parseConfigFromSearch()` in `widget.html`
- Document: `CLAUDE.md` parameter table + `README.md`
- Test: `tests/unit.test.js` (parseConfigFromSearch describe block)

**New admin feature:**
- Implementation: inline `<script>` in `generator.html`
- No existing test infrastructure for generator (see CONCERNS.md)
